`<Studio>` の左サイドバーは、44 px の垂直アイコンレールに加えて、
4 つの相互排他的なワークスペース（`insert`、`layer`、`image`、`text`）の
いずれか 1 つをホストする 280 px のモジュールパネルを
提供します。ホストはレール + パネルを無償で利用できます。
プラグインは 5 つの登録面を通じて任意のモジュールを拡張できます。
利用者は `<Studio messages>` とインスタンスごとの store を介して、
文言・比率・永続化された状態をオーバーライドできます。

## モジュール

| モジュール | 役割 | 永続化される状態 |
| :--- | :--- | :--- |
| `insert` | ドラッグ＆ドロップ式のコンポーネントライブラリ — セクション分け、検索可能、グリッド/リスト表示。Puck の `<Puck.Components />` に委譲します。 | `componentViewMode`、`insertSectionsExpanded`、`drawerSearch`（一時的） |
| `layer` | アクティブなページの要素ツリー（`<Puck.Outline />`）の上に、（ホストが供給する）ページ一覧を、水平ドラッグハンドルで分割して表示します。 | `pagesExpanded`、`layerSplitRatio`、`outlineExpanded` |
| `image` | アセットライブラリ — フィルターストリップ（All / Images / Videos / Audio）、検索、ドラッグ＆ドロップアップロード、クリックで挿入、アセットごとのオーバーフローメニュー。`@anvilkit/plugin-asset-manager` にブリッジします。 | `assetCategoryFilter` |
| `text` | コピーライティングのスニペットライブラリ — カテゴリ、検索、キャンバス上で選択された Text 要素へのクリック挿入。 | `copyCategoryFilter` |

アクティブなモジュールは `<Studio>` インスタンスごとに
`anvilkit-ui-${storeId}` の下で永続化されます。同一ページに 2 つの
`<Studio>` インスタンスをマウントする場合は、それぞれ異なる
`storeId` 属性を指定してください。

## モジュール切り替えの契約

- **非アクティブ**なモジュールのレールアイコンをクリックすると、そのモジュールに切り替わります。
- **すでにアクティブ**なモジュールのレールアイコンをクリックすると、
  パネルが折りたたまれます（レールは表示されたままです）。
- パネルヘッダーの `×` をクリックすると、アクティブなモジュールを
  変更せずに折りたたまれます — いずれかのレールアイコンを
  再クリックすると復元されます。
- パネル内のどこかにフォーカスがある状態で `Esc` を押すと折りたたまれ、
  フォーカスはアクティブなレールアイコンに戻ります。

## プラグイン拡張面

プラグインは `register()` で型付けされた `StudioPluginContext` を
受け取ります。各 `register*` ヘルパーは `unregister()` 関数を返し、
プラグインはそれを `onDestroy` ライフサイクルフックから
呼び出すべきです。

| 拡張面 | モジュール | 用途 |
| :--- | :--- | :--- |
| `ctx.registerInsertSection(section)` | `insert` | Puck コンポーネントに対するカスタム述語を持つ、新しい折りたたみ可能なセクションを追加する |
| `ctx.registerLayerQuickAdd(item)` | `layer`（Layers） | 「+」のクイック追加ポップオーバーにエントリを追加する |
| `ctx.registerAssetSource(source)` | `image` | アセット一覧 + アップロード + ミューテーションの契約を提供する |
| `ctx.registerAssetAction(action)` | `image` | アセットごとの `…` オーバーフローメニューにエントリを追加する |
| `ctx.registerCopySnippetPack(pack)` | `text` | スニペットのセットを、必要に応じてロケールごとに提供する |

`apps/demo/lib/puck-demo.ts` のデモには、独自に書く前に読んでおく
価値のある 2 つの参照プラグインが付属しています。

- `demoCopySnippetPlugin` は `ctx.registerCopySnippetPack` を介して
  英語のスニペットパックを登録します。12 個のエントリが `basic`
  / `brand` カテゴリに分割されており、
  `StudioCopySnippetPack` の構造リファレンスとして役立ちます。
- `demoLayerQuickAddPlugin` は `ctx.registerLayerQuickAdd` を介して
  単一のレイヤークイック追加を登録します。ライブの Puck スナップショットを
  読み取り、その後 `insert` アクションを派生（dispatch）します — これは
  クイック追加を提供するあらゆるプラグインが従うべき構造と
  同じです。

アセットソースの契約は
[`@anvilkit/plugin-asset-manager`](/ja/api/plugin-asset-manager/) にあります。
その `studio-asset-source.ts` アダプターは、
`StudioAssetSource` を実装するための正規のリファレンスです。

## i18n カタログ

すべてのレールツールチップ、パネルタイトル、検索プレースホルダー、
空状態の文言、オーバーフローメニュー項目は i18n provider を通じて解決されます。
デフォルトの英語カタログは
[`editor-i18n-context.tsx`](https://github.com/ancyloce/anvilkit-studio/blob/main/packages/core/src/state/editor-i18n-context.tsx)
の `studio.module.*` 名前空間の下で提供されます。

オーバーライドは `<Studio messages>` 経由で渡します。

```tsx title="app/editor/page.tsx"
<Studio
	puckConfig={config}
	data={data}
	messages={{
		"studio.module.insert.name": "Components",
		"studio.module.layer.name": "Pages",
	}}
/>
```

### メッセージの解決

解決順序（優先度 高 → 低）：要求されたキーのオーバーライド、
要求されたキーのデフォルト、呼び出し元が供給する `fallback`、
キーそのもの。網羅的な解決順序テストは
`packages/core/src/state/__tests__/editor-i18n-context.alias.test.tsx` にあります。

<Callout type="info" title="削除：レガシーなタブエイリアス">
	v1 以前のサイドバーには `studio.tab.insert` / `studio.tab.outline`
	キー（v1 の `studio.module.insert.name` /
	`studio.module.layer.name` へのエイリアス）が付属していました。これらのエイリアスとレガシーキーは
	いずれも **削除** されました。`studio.module.*` キーを直接オーバーライドしてください。
</Callout>

## CSS 変数

サイドバーは既存のすべての `--ak-studio-*` トークンを再利用し、さらに
chrome が iframe テーマ同期を通じてブリッジする 2 つの新しい
レイアウトトークンを使用します。

| トークン | デフォルト | 用途 |
| :--- | :--- | :--- |
| `--ak-studio-rail-width` | `44px` | 常時表示されるアイコンレールの幅 |
| `--ak-studio-panel-width` | `280px` | 折りたたまれていないときのモジュールパネルの幅 |

`:root`（または `<Studio>` の任意の祖先）でいずれかをオーバーライドすると、
chrome をフォークすることなくレール / パネルを再スキンできます。

## レガシーな 2 タブシェルからの移行

`EditorTab` は `"insert" | "outline"` から `"insert" |
"layer" | "image" | "text"` に拡張されます。`editor-ui-store` の persist バージョンは
`2` に上がります。hydrate のマイグレーションは、初回ロード時に一度だけ
レガシーな `"outline"` → `"layer"` を書き換えます。
`EditorTab === "outline"` で絞り込んでいる利用者は更新する必要があります。

レガシーな `studio.tab.*` キーを接続していたホストは、エイリアスマップを
通じて引き続き動作します（1 リリース分のウィンドウ）。

## API リファレンス

- [`StudioProps.messages`](/ja/api/core/interfaces/studioprops/) — エディターの i18n provider に転送される i18n オーバーライド
- [`StudioPagesSource`](/ja/api/core/interfaces/studiopagessource/) — `layer` モジュールのページ一覧を埋めるためにホストが実装する契約
- [`StudioPluginContext`](/ja/api/core/interfaces/studioplugincontext/) — 登録ヘルパー（`registerInsertSection`、`registerLayerQuickAdd`、`registerAssetSource`、`registerAssetAction`、`registerCopySnippetPack`）
