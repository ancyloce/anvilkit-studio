## Functions

| Function | Description |
| ------ | ------ |
| [createCanvasPresence](functions/createCanvasPresence.md) | Bridge between a `y-protocols/awareness` instance and the canvas presence contract. Ported from `plugin-collab-yjs`'s `createAwarenessBridge`, minus the metrics coupling: |
| [createCanvasYjsBinding](functions/createCanvasYjsBinding.md) | Bind a canvas [SceneStoreApi](#) to a `Y.Doc` (I3-1 prototype). Mirrors `plugin-collab-yjs`'s `createYjsAdapter`, adapted for `CanvasIR` and a zustand store rather than the PageIR snapshot contract. |
| [decodeCanvasIR](functions/decodeCanvasIR.md) | Parse + validate a remote payload. Unlike the plugin's `decodeIR` (which only checks `version === "1"`), this runs canvas-core's full [CanvasIRSchema](https://github.com/ancyloce/anvilkit-canvas-core/blob/main/src/ir/validators.ts) — a hostile or corrupt peer cannot inject a structurally-invalid IR into the local scene. Throws on invalid input; callers in observers must wrap this in try/catch (never throw out of a Yjs observer). |
| [encodeCanvasIR](functions/encodeCanvasIR.md) | Serialize a [CanvasIR](https://github.com/ancyloce/anvilkit-canvas-core/blob/main/src/ir/types.ts) to a stable JSON string with sorted object keys. Two replicas observing the same logical IR must produce byte-identical Y.Map values; otherwise last-writer-wins would flap on key-order differences alone. Mirrors `plugin-collab-yjs`'s `encodeIR`. |
| [sanitizeDisplayName](functions/sanitizeDisplayName.md) | Strip ASCII control characters (0x00–0x1f) and DEL (0x7f) from a display name, then cap to [MAX\_DISPLAY\_NAME\_LENGTH](variables/MAX_DISPLAY_NAME_LENGTH.md). Filters by code point to avoid a control-character regex literal. Does NOT escape HTML — hosts rendering the name into `innerHTML` must still escape; this only blunts control-character injection. |
| [useCanvasPresence](functions/useCanvasPresence.md) | Subscribe to validated remote peer presence (cursors / selections) from the nearest [CanvasPresenceContext](variables/CanvasPresenceContext.md). Returns `[]` when no provider is mounted. This is the data seam for the deferred collab UI (I3-1): the presence layer (`<RemoteCursors>` / `<RemoteSelections>`) can read it once cursor/selection rendering ships, without changing the binding. |
