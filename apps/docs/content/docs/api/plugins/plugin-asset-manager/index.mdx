## Classes

| Class | Description |
| ------ | ------ |
| [AssetResolutionError](classes/AssetResolutionError.md) | Error thrown when an `asset://` reference cannot be resolved safely. |
| [AssetSourceError](classes/AssetSourceError.md) | Error for async catalog operations: host `dataSource` failures, folder mutations, and external `AssetSourceProvider` calls (e.g. Unsplash). Carries retry metadata so callers can drive backoff (`./retry`) without re-deriving it: `retryable` gates whether a retry is attempted at all, `status` mirrors an HTTP status (429, 401, …), and `retryAfterMs` is sourced from a `Retry-After` / `X-Ratelimit-Reset` header when present. |
| [AssetValidationError](classes/AssetValidationError.md) | Error thrown when an upload adapter returns an invalid result. |

| [PersistedUploadSession](interfaces/PersistedUploadSession.md) | The serializable subset of an [UploadSession](interfaces/UploadSession.md) that the M2 session store persists so an interrupted upload can resume after a reload. MUST be JSON / structured-clone safe end-to-end — it round-trips through `localStorage`. |
| [RequiredCsp](interfaces/RequiredCsp.md) | Computed CSP directive sources needed for configured asset backends. |
| [RequiredCspOptions](interfaces/RequiredCspOptions.md) | Inputs describing which asset backends need CSP allowances. |
| [ResumableUploadAdapter](interfaces/ResumableUploadAdapter.md) | Session-based upload backend. Contrast with [UploadAdapter](type-aliases/UploadAdapter.md), which is a single `File → UploadResult` call. Implementations own the four lifecycle steps; the adapter-agnostic runner (M3) drives them and handles slicing, per-part retry, progress, resume, and abort. |
| [ResumableUploadConfig](interfaces/ResumableUploadConfig.md) | Resumable / multipart upload configuration on [AssetManagerOptions](interfaces/AssetManagerOptions.md). |

| [UnsplashTheme](interfaces/UnsplashTheme.md) | - |
| [UnsplashThemeConfig](interfaces/UnsplashThemeConfig.md) | Theme configuration for the built-in Unsplash provider. |
| [UploadAdapterOptions](interfaces/UploadAdapterOptions.md) | Options passed to an [UploadAdapter](type-aliases/UploadAdapter.md). Currently carries an optional `AbortSignal` so a cancelled upload batch can short-circuit the adapter (network request, image decode). Adapters may ignore it. |
| [UploadPart](interfaces/UploadPart.md) | One contiguous chunk of the source file handed to [ResumableUploadAdapter.uploadPart](interfaces/ResumableUploadAdapter.md#uploadpart). |
| [UploadResult](interfaces/UploadResult.md) | Normalized asset row returned by upload adapters and registry mutations. |
| [UploadSession](interfaces/UploadSession.md) | Backend handle for an in-progress multipart upload, returned by [ResumableUploadAdapter.begin](interfaces/ResumableUploadAdapter.md#begin). |
| [UploadSessionStore](interfaces/UploadSessionStore.md) | Persistence seam for resumable uploads. The runner loads a prior session before [ResumableUploadAdapter.begin](interfaces/ResumableUploadAdapter.md#begin) (so an interrupted upload can resume), saves progress as parts complete, and clears it once the upload finishes or is abandoned. Keyed internally by a stable file fingerprint (name + size + lastModified), so the same picked file resumes while a different file does not. |

## Type Aliases

| Type Alias | Description |
| ------ | ------ |
| [AssetDeletedHook](type-aliases/AssetDeletedHook.md) | Lifecycle hook fired when an asset is deleted through the asset source, with the record as it existed at deletion time. Lets a host release backend objects tied to the upload (e.g. delete the stored S3 object, revoke a `blob:` URL). Awaited; throwing rejects the delete. |
| [AssetKind](type-aliases/AssetKind.md) | Library-management kind union surfaced by `AssetRegistry.search` and the sidebar filter row. Mirrors `@anvilkit/core/types`' `StudioAssetKind` shape so the same filter values flow through both the headless registry and the Studio surface without translation. |
| [AssetManagerEventName](type-aliases/AssetManagerEventName.md) | Supported asset-manager event names. |
| [AssetRegistryListener](type-aliases/AssetRegistryListener.md) | Listener invoked after every registry mutation (`register` / `delete` / `rename` / `replace` / `setTags`). The `studio-asset-source` adapter wires its own listener set onto the registry so the sidebar re-runs `list()` whenever an asset changes. |
| [AssetResolutionErrorCode](type-aliases/AssetResolutionErrorCode.md) | - |
| [AssetSortField](type-aliases/AssetSortField.md) | - |
| [AssetSourceErrorCode](type-aliases/AssetSourceErrorCode.md) | Failure codes raised by the async data-source / folder / external-provider surfaces (PRD 0002 §3.6). Distinct from [AssetValidationError](classes/AssetValidationError.md) (upload validation) and [AssetResolutionError](classes/AssetResolutionError.md) (resolver domain) — in particular `ASSET_NOT_FOUND` belongs to the resolver, never here. |
| [AssetSourceId](type-aliases/AssetSourceId.md) | Stable source identifiers. Branded so a typo is a type error while hosts can still introduce their own ids (e.g. `"brandfolder"`). |
| [AssetSourceStatus](type-aliases/AssetSourceStatus.md) | Loading/mutation status surfaced through [AssetDataSource.subscribeStatus](interfaces/AssetDataSource.md#subscribestatus). |
| [FolderId](type-aliases/FolderId.md) | Canonical id of a folder. `null` (never a real id) denotes the root. |
| [JsonValue](type-aliases/JsonValue.md) | Any value that survives `JSON.stringify` → `JSON.parse` (and `structuredClone`) unchanged. Used to type persisted continuation data so the M2 session store can round-trip it through `localStorage` without silently dropping functions, symbols, `BigInt`, class instances, or cyclic references. |
| [ReplacePayload](type-aliases/ReplacePayload.md) | Replacement payload for [AssetDataSource.replace](interfaces/AssetDataSource.md#replace). Kept `File`-only for parity with core's `StudioAssetSource.replace` boundary; programmatic `UploadResult` replacement stays on the registry. |
| [TransformResolver](type-aliases/TransformResolver.md) | Host-pluggable mapping from an asset + transform spec to a derivative URL. Return `undefined` to fall back to the asset's original URL (e.g. the transform isn't supported, or the asset isn't an image). The returned URL is re-validated through the same trust boundary as any other resolved asset URL. |
| [UploadAdapter](type-aliases/UploadAdapter.md) | - |
| [ValidateUploadResultOptions](type-aliases/ValidateUploadResultOptions.md) | Subset of [AssetManagerOptions](interfaces/AssetManagerOptions.md) consulted by the trust boundary. `urlAllowlist` is intentionally absent — Phase 4 replaced the raw allowlist with the two typed flags below. |

Not for production; use `s3PresignedAdapter` or a custom `UploadAdapter` there. |

| [getRequiredCsp](functions/getRequiredCsp.md) | Compute CSP origins required by the configured asset upload backends. |

| [inMemoryUploader](functions/inMemoryUploader.md) | In-memory upload adapter for demos and tests. Returns a `blob:` URL that the consumer renders directly, so the object URL intentionally lives for the page lifetime — it cannot be revoked while the asset is still referenced. |
| [isResumableAdapter](functions/isResumableAdapter.md) | Narrow an upload backend to a [ResumableUploadAdapter](interfaces/ResumableUploadAdapter.md). The single-shot [UploadAdapter](type-aliases/UploadAdapter.md) is a function; the resumable adapter is an object with the four lifecycle methods, so a `typeof === "object"` plus method check distinguishes them unambiguously. |
| [parseAssetReference](functions/parseAssetReference.md) | Parse an `asset://<id>[?transform]` reference. Returns `null` for any non- asset URL or an empty id. Malformed transform params are ignored rather than failing the parse (references can be host- or tamper-supplied). |
| [resolveAssets](functions/resolveAssets.md) | Resolve all asset references found in a Page IR tree. |
| [resolveFolderId](functions/resolveFolderId.md) | Normalize a folder-id input to the canonical root convention: `undefined` and `null` both collapse to `null` (root); a real id is returned unchanged. This is the only folder-id normalization helper — there is no `__root__` sentinel to expand or collapse. |
| [uploadAsset](functions/uploadAsset.md) | Validate and upload a single file through the configured plugin runtime. |
| [validateUploadResult](functions/validateUploadResult.md) | Validate and sanitize an upload adapter result before registry insertion. |
