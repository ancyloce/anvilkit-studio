| &lt;a id="maxbytes"&gt;&lt;/a&gt; `maxBytes?` | `readonly` | `number` | Maximum **raw file size** in bytes (default 1 MB). This bounds the input file, NOT the emitted `data:` URL: base64 encoding inflates the payload ~4/3, so a file at the cap becomes a ~1.33× larger string that is held in memory and embedded inline in the registry / IR / exported page (C3).
