## Classes

| Class | Description |
| ------ | ------ |
| [DebouncedAdapterDestroyedError](classes/DebouncedAdapterDestroyedError.md) | Wrap a `SnapshotAdapter` so that bursts of `save()` calls within `ms` coalesce into a single underlying write. The latest IR wins; every caller that issued a save during the window receives the resolved id (or the rejection) of that single flush. |
| [InvalidAdapterOptionsError](classes/InvalidAdapterOptionsError.md) | §4.2.4 — typed construction-time failure for [createYjsAdapter](functions/createYjsAdapter.md). |
| [SnapshotCorruptedError](classes/SnapshotCorruptedError.md) | The record is internally inconsistent — the payload failed to decode (corrupt / schema-drifted), or metadata is present without its payload (which, because retention deletes both atomically in one synchronous transaction, indicates corruption, not eviction). |
| [SnapshotNotFoundError](classes/SnapshotNotFoundError.md) | No snapshot with this id exists in the shared Y.Doc. |
| [SnapshotPrunedError](classes/SnapshotPrunedError.md) | The snapshot was evicted by the `maxSnapshots` retention cap. This is best-effort and **same-session**: it is raised when THIS adapter instance performed the eviction (it tracks its own pruned ids). Cross-peer or post-reload, an evicted id is indistinguishable from one that never existed (retention deletes meta+payload atomically) and surfaces as [SnapshotNotFoundError](classes/SnapshotNotFoundError.md); a host history UI should pre-filter requests against the retained `list()`. Recoverable: hide / disable the revision rather than treat it as corruption. |

| [PersistenceOptions](interfaces/PersistenceOptions.md) | Opt-in cross-tab persistence (L5). The default `createYjsAdapter` call uses no persistence — outbound Y.js updates flow through the host transport (websocket, hocuspocus, etc.) only. Enabling `indexedDb` adds a durable queue so edits made while offline replay on reconnect. Enabling `broadcastChannel` adds same-origin cross-tab sync so two tabs of the same app see each other's edits without round-tripping through the transport. |
| [PolicyViolation](interfaces/PolicyViolation.md) | Reasons a save or dispatch was rejected by `policy.canEdit`. |
| [PresenceActivity](interfaces/PresenceActivity.md) | Activity / typing metadata for a peer. Lets hosts render "Alice is typing…" affordances. Both fields are optional and individually validated. |
| [PresenceViewport](interfaces/PresenceViewport.md) | Peer viewport — scroll offset, zoom factor, and an optional visible rect. Lets remote clients render "where is this peer looking" follow-along UI. All numbers are finite; `zoom`/`width`/`height` are clamped to sane bounds by [validatePresenceViewport](functions/validatePresenceViewport.md). |
| [PropGuardOptions](interfaces/PropGuardOptions.md) | Y3/§4.1.3 — per-prop decode bounds. All four are independent ceilings checked while decoding a single prop value; the first one exceeded drops that prop and records its reason. Defaults are deliberately generous so legitimate PageIR (rich-text, nested layout props, item lists) is never clamped. |
| [RichPresenceState](interfaces/RichPresenceState.md) | Versioned, backward-compatible superset of PresenceState (§4.2.5). Every richer field is optional, so a plain `PresenceState` is a valid `RichPresenceState` and vice versa. Multi-select reuses the existing `selection.nodeIds` array (which already permits many ids); the additions are viewport, focused-block, activity, and the schema `version`. |
| [RichSnapshotAdapterPresence](interfaces/RichSnapshotAdapterPresence.md) | Richer-typed view of SnapshotAdapterPresence (§4.2.5). Same runtime object — `update` accepts and `onPeerChange` emits [RichPresenceState](interfaces/RichPresenceState.md), so hosts can set and read the versioned multi-select / viewport / focused-block / typing fields while the base cursor + selection contract keeps working. Structurally assignable to `SnapshotAdapterPresence` (the extra fields are optional). |
| [SharedTypeSupport](interfaces/SharedTypeSupport.md) | Shape of [SHARED\_TYPE\_SUPPORT](variables/SHARED_TYPE_SUPPORT.md). |
| [SnapshotPersistenceAdapter](interfaces/SnapshotPersistenceAdapter.md) | §4.2.2 — pluggable backend storage for saved snapshots ("server-grade snapshot persistence"). The in-`Y.Doc` snapshot store is bounded by [CreateYjsAdapterOptions.maxSnapshots](interfaces/CreateYjsAdapterOptions.md#maxsnapshots) and is the source of truth; this is an OPTIONAL durable sink an embedder wires to a real backend (Postgres, S3, a `fetch` endpoint, IndexedDB object store …) so full-state snapshots survive eviction and reloads. |
| [SnapshotPersistenceOptions](interfaces/SnapshotPersistenceOptions.md) | §4.2.2 — configuration bag for [CreateYjsAdapterOptions.snapshotPersistence](interfaces/CreateYjsAdapterOptions.md#snapshotpersistence). Bundles the [SnapshotPersistenceAdapter](interfaces/SnapshotPersistenceAdapter.md) with an optional reversible payload transform (the encryption-at-rest seam) and a fault sink. |
| [UndoController](interfaces/UndoController.md) | Local undo/redo surface exposed by [YjsSnapshotAdapter](interfaces/YjsSnapshotAdapter.md) when [CreateYjsAdapterOptions.undo](interfaces/CreateYjsAdapterOptions.md#undo) is provided. All methods are always present; when undo was not enabled they are inert (`canUndo`/`canRedo` return `false`, `undo`/`redo`/`clearUndo` are no-ops, `onUndoStackChange` never fires). |
| [UndoOptions](interfaces/UndoOptions.md) | Local undo/redo configuration (§4.1.1). Maps the report's `{ scopes, trackedOrigins, captureTimeout }` shape onto this adapter: the *scope* is fixed (the adapter owns it — the native `:tree` map or the legacy whole-document map), so only the origin filter and the capture window are tunable here. |
| [ValidationFailure](interfaces/ValidationFailure.md) | Reasons a remote update was rejected after `validateRemoteIR` ran. |
| [YjsSnapshotAdapter](interfaces/YjsSnapshotAdapter.md) | Yjs-specific extension of `SnapshotAdapter` that exposes the conflict-diagnostics event surface, the connection-state contract, and the force-resync action. `createYjsAdapter` returns this narrow type so hosts can subscribe to overlap events and render sync UI without dropping into Yjs internals. |

## Variables

| Variable | Description |
| ------ | ------ |
| [MAX\_DISPLAY\_NAME\_LENGTH](variables/MAX_DISPLAY_NAME_LENGTH.md) | Structured validation for `PresenceState` payloads. Replaces the scattered ad-hoc type guards previously inline in `yjs-adapter.ts`. |
| [MAX\_NODE\_ID\_LENGTH](variables/MAX_NODE_ID_LENGTH.md) | Upper bound on the length of a single node-id / focused-block-id string. Mirrors [MAX\_DISPLAY\_NAME\_LENGTH](variables/MAX_DISPLAY_NAME_LENGTH.md) in spirit — keeps a peer from smuggling a megabyte string through an id field. |
| [MAX\_SELECTION\_IDS](variables/MAX_SELECTION_IDS.md) | Upper bound on the number of node ids carried in a multi-select presence payload. A hostile/buggy peer that "select-all"s a huge document must not be able to flood every other client's awareness cache with an unbounded id array — excess ids beyond this cap are dropped (the selection is truncated, not the whole state rejected). |
| [MAX\_VIEWPORT\_ZOOM](variables/MAX_VIEWPORT_ZOOM.md) | - |
| [MIN\_VIEWPORT\_ZOOM](variables/MIN_VIEWPORT_ZOOM.md) | Clamp range for `viewport.zoom`. Values outside the range are clamped (not rejected) so a peer mid-gesture with a transient out-of-range zoom still produces a usable viewport. |
| [PRESENCE\_SCHEMA\_VERSION](variables/PRESENCE_SCHEMA_VERSION.md) | Current presence payload schema version (§4.2.5). |
| [SHARED\_TYPE\_SUPPORT](variables/SHARED_TYPE_SUPPORT.md) | Public, machine-checkable descriptor of which Yjs shared types `createYjsAdapter` binds. See the module doc-comment for the rationale. |
