## Functions

| Function | Description |
| ------ | ------ |
| [CollabPresenceAnnouncer](functions/CollabPresenceAnnouncer.md) | Polite a11y live region that announces collaborator join/leave changes to screen readers (M3 / review ¬ß4.2.3). The aggregate `aria-label` on `<PeerAvatarStack>` exposes the *current* count, but it does not narrate the *transition* when a collaborator arrives or departs ‚Äî this region does. |
| [CollabPresencePublisher](functions/CollabPresencePublisher.md) | Opt-in local presence publisher (review ¬ßC4). |
| [CollabRoomBar](functions/CollabRoomBar.md) | - |
| [CollabSettingsPopover](functions/CollabSettingsPopover.md) | - |
| [CollabUII18nProvider](functions/CollabUII18nProvider.md) | - |
| [CollabUIProvider](functions/CollabUIProvider.md) | - |
| [CollabUndoRedoControls](functions/CollabUndoRedoControls.md) | Toolbar pair of undo / redo buttons wired to the adapter's local undo/redo stack via [useCollabUndoState](functions/useCollabUndoState.md) (¬ß4.1.2). Each button's disabled state is driven straight from the adapter's `canUndo()` / `canRedo()` getters ‚Äî when undo is disabled (or the adapter predates the controller) both buttons render disabled and stay inert. |
| [conflictKey](functions/conflictKey.md) | Stable identity for a conflict event. Composite (not the bare ISO `at`) because two overlap conflicts can share a timestamp within one tick ‚Äî see M1 / review ¬ßC6. Used both as the sonner toast id and as the `dismissConflict` key, so dismissing one never drops a co-timestamped sibling unacknowledged. |
| [ConflictNoticeCenter](functions/ConflictNoticeCenter.md) | - |
| [ConnectionRecoveryBanner](functions/ConnectionRecoveryBanner.md) | Proactive disconnect-recovery banner (H4 / ¬ß4.1.4). Reads connection status via [useCollabStatus](functions/useCollabStatus.md) and renders ONLY while the session is degraded (`offline`/`reconnecting`/`error`) ‚Äî otherwise returns `null`, so a host can mount it unconditionally above the editor chrome. |
| [createCollabPlugin](functions/createCollabPlugin.md) | Build a single StudioPlugin that delivers both yjs data sync *and* the collab UI (presence cursors, conflict toasts, collaborator avatar stack) in one factory call. |
| [ForceResyncDialog](functions/ForceResyncDialog.md) | - |
| [PeerAvatarStack](functions/PeerAvatarStack.md) | Stacked collaborator avatars. Reads only identity + peers context, so it does not re-render on status / conflict / cursor churn (review ¬ßC2). |
| [PresenceLayer](functions/PresenceLayer.md) | Mounts spring-animated cursors and selection rings for every remote peer in the active collab room. |
| [SyncActivityIndicator](functions/SyncActivityIndicator.md) | - |
| [useCollabAdapter](functions/useCollabAdapter.md) | The live YjsSnapshotAdapter the provider was mounted with. |
| [useCollabConflictQueue](functions/useCollabConflictQueue.md) | Conflict queue plus its mutators. Reads only `ConflictsContext`. |
| [useCollabConflicts](functions/useCollabConflicts.md) | The retained window of recent ConflictEvents. Reads only `ConflictsContext`; use [useCollabConflictQueue](functions/useCollabConflictQueue.md) when you also need the dismiss / clear mutators. |
| [useCollabContext](functions/useCollabContext.md) | Back-compat composite hook. Returns the full [CollabUIContextValue](interfaces/CollabUIContextValue.md). Prefer the narrow hooks (`useCollabStatus`, `useCollabPeers`, ‚Ä¶) in components ‚Äî this hook re-renders on **any** collab state change by design. |
| [useCollabCursorVisibility](functions/useCollabCursorVisibility.md) | Shared remote-cursor visibility state (review ¬ßC3 / ¬ß4.3). The bundled settings popover writes it; the bundled presence overlay reads it. |
| [useCollabIdentity](functions/useCollabIdentity.md) | Identity tuple `{ self, updateSelf }`. Reads only `IdentityContext`, so consumers do not re-render on cursor / status / conflict churn. |
| [useCollabMetrics](functions/useCollabMetrics.md) | Poll `adapter.metrics()` on an interval (review ¬ßC5 / ¬ß4.5). |
| [useCollabPeerIdentities](functions/useCollabPeerIdentities.md) | Roster-only peer identities (id/displayName/color). The returned array reference only changes when the *set* of identities changes ‚Äî not on cursor / selection frames (F5) ‚Äî so identity-only consumers like `<PeerAvatarStack>` skip cursor-churn re-renders. Excludes the local peer (mirrors `useCollabPeers()`). |
| [useCollabPeers](functions/useCollabPeers.md) | The remote peers' full presence frames (identity + cursor + selection), excluding the local peer. Re-renders on cursor/selection churn ‚Äî prefer [useCollabPeerIdentities](functions/useCollabPeerIdentities.md) for identity-only consumers. |
| [useCollabSelf](functions/useCollabSelf.md) | The local peer's current identity (`self`). Reads only `IdentityContext`. |
| [useCollabStatus](functions/useCollabStatus.md) | Current adapter ConnectionStatus. Reads only `StatusContext`, so consumers don't re-render on peer / cursor / conflict churn. |
| [useCollabUndoState](functions/useCollabUndoState.md) | Subscribe to the adapter's local undo/redo stack (¬ß4.1.2). Mirrors the `UndoController` surface exposed by YjsSnapshotAdapter: it backs `canUndo`/`canRedo` with `useSyncExternalStore` over `adapter.onUndoStackChange`, so a toolbar re-reads enabled-state the instant a tracked edit (or an `undo()`/`redo()`) mutates the stack ‚Äî without a stale-paint window. |
