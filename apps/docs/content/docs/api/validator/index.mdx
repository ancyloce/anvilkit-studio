| [validatePageRootProps](functions/validatePageRootProps.md) | Pure, isomorphic gateway that validates a page's `root.props` against the canonical PageRootSchema from `@anvilkit/schema`. This is the narrow, metadata-only check used by draft saves and settings edits. |
| [validatePublishRequest](functions/validatePublishRequest.md) | Validate the body of a `POST /api/pages/publish` request (`{ id?, slug?, data: PuckPageData }`). Publishing goes live, so the complete payload is validated via [validatePuckPageData](functions/validatePuckPageData.md). |
| [validatePuckPageData](functions/validatePuckPageData.md) | Validate a *complete* Puck page document (`{ root, content, zones? }`) — the check a publish must pass before the payload is written to storage. Delegates `root.props` to the same PageRootSchema as [validatePageRootProps](functions/validatePageRootProps.md) and additionally asserts the structural shape the renderer depends on. |
| [validateSaveDraftRequest](functions/validateSaveDraftRequest.md) | Validate the body of a `POST /api/pages/draft` request (`{ slug?, title?, data: { root: { props } } }`). A draft only commits page metadata, so this validates `data.root.props` and tolerates arbitrary `content` (persisted verbatim). |
