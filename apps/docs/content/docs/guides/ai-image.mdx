---
title: AI Image plugin
description: Provider-delegated text-to-image, variation, inpaint, and background-removal jobs for AnvilKit Canvas Studio via @anvilkit/plugin-ai-image.
---

`@anvilkit/plugin-ai-image` wraps a host-supplied `AiImageProvider` so the
canvas editor can issue text-to-image, variation, inpaint, background-removal,
and upscale jobs through one typed contract. The plugin **never** talks to an AI
service directly — no credentials, model ids, or endpoints cross the plugin
boundary. The surrounding building blocks (abort/retry/poll job client, mock
provider, mask editor, post-processing pipeline, canvas commit helper, and an
optional React sidebar panel) ship as separate subpath exports.

## 1. Install

```sh
pnpm add @anvilkit/plugin-ai-image react react-dom @puckeditor/core
```

Peer dependencies: `react >=19.0.0`, `react-dom >=19.0.0`,
`@puckeditor/core ^0.21.3`, `@anvilkit/ui` (for the React panel), and
`konva ^9 || ^10` + `react-konva ^19` (for the `./mask` and `./react` Konva
surfaces). The headless `.` entry — factory, job client, post-process, commit —
does not require Konva.

Subpath exports:

- `@anvilkit/plugin-ai-image` — `createAiImagePlugin`, `createAiJobClient`, `commitImageReplace`, mask exporter utilities, and all wire-shape types.
- `@anvilkit/plugin-ai-image/react` — `AiImagePanel`, `useAiImage`, `createAiImageSidebarPlugin`.
- `@anvilkit/plugin-ai-image/mask` — `MaskEditorLayer`, `useMaskStrokes`, mask types.
- `@anvilkit/plugin-ai-image/post-process` — `createPostProcessPipeline` + image helpers.
- `@anvilkit/plugin-ai-image/commit` — `commitImageReplace`.
- `@anvilkit/plugin-ai-image/mock` — `createMockAiImageProvider` for tests and demos.

## 2. The provider contract

`AiImagePluginOptions` carries a single required `provider`. A provider is an
async function that receives a typed request, a layer context, and run options
(including an `AbortSignal`), and resolves an `AiImageJobResult`:

```ts
import { createAiImagePlugin } from "@anvilkit/plugin-ai-image";
import type { AiImageProvider } from "@anvilkit/plugin-ai-image";

const provider: AiImageProvider = async (request, context, options) => {
  // Call your AI service of choice (Replicate, OpenAI, self-hosted SD, …).
  // Honour `options?.signal` for cancellation and return an AiImageJobResult.
  const res = await fetch("/api/ai/image", {
    method: "POST",
    body: JSON.stringify({ request, context }),
    signal: options?.signal,
  });
  return (await res.json()) as Awaited<ReturnType<AiImageProvider>>;
};

const aiImage = createAiImagePlugin({ provider });
```

For tests and demos, `createMockAiImageProvider()` from the `./mock` subpath
returns deterministic results without a network call.

## 3. Mounting the UI

<Callout type="warn" title="The bare factory is headless">
`createAiImagePlugin` validates the provider and exposes an imperative
`submit()` method, but registers an **empty** Studio hook block — on its own it
contributes no lifecycle hooks or UI. Mount the panel via
`@anvilkit/plugin-ai-image/react` (`AiImagePanel` / `createAiImageSidebarPlugin`)
or build your own UI on top of the `useAiImage` hook.
</Callout>

## 4. Job client, masks, and commit

- **Job client** — `createAiJobClient` adds abort/retry/poll around a provider;
  it surfaces `RetryableError` for transient failures.
- **Masks** — `drawMask`, `rasterizeMaskToDataUrl`, and `createMaskToAssetExporter`
  turn editor strokes into a mask asset for inpaint jobs; `dataUrlToFile` bridges
  to upload adapters.
- **Commit** — `commitImageReplace` writes a generated image back onto the
  canvas layer that requested it.
- **Post-process** — `createPostProcessPipeline` (subpath-only, to keep the main
  bundle lean) compresses and thumbnails results before upload.
