`@anvilkit/plugin-canvas-studio`는 Puck 기반의 `<Studio>` 셸 안에 전체 화면
**Canvas Studio** 편집 표면을 마운트합니다. 헤더 액션으로 Puck 페이지 모드와
캔버스 오버레이 사이를 전환합니다. 닫을 때 플러그인은 호스트가 제공한
어댑터를 통해 `CanvasIR`을 영속화하고, 각 아트보드를 미리보기로
래스터화하며, 원본 Puck `DesignBlock`을 패치하여 페이지가 그 썸네일을
렌더링하도록 합니다. 또한 디자인 블록을 위한 레이어 빠른 추가 항목과,
페이지가 디자인 id로 아트보드 미리보기를 참조할 수 있는 `design://` 에셋
리졸버도 등록합니다.

## 1. 설치

```sh
pnpm add @anvilkit/plugin-canvas-studio @anvilkit/canvas-core @anvilkit/canvas-editor react react-dom @puckeditor/core
```

피어 의존성: `@anvilkit/canvas-core`, `@anvilkit/canvas-editor`,
`@puckeditor/core ^0.21.3`, `react >=19.0.0`,
`react-dom >=19.0.0`.

<Callout type="warn" title="에디터 스타일시트 가져오기">
Tailwind 유틸리티와 부모 문서 CSS는 캔버스에 **도달하지 않습니다** — 에디터는
자체 완결형 스타일시트를 함께 제공합니다. 호스트는 그것을 한 번 가져와야
하며, 예를 들어 앱 진입점에서:

```ts
import "@anvilkit/canvas-editor/styles.css";
```
</Callout>

## 2. 최소 호스트

```ts
import { Studio } from "@anvilkit/core";
import {
  createCanvasStudioPlugin,
  localStorageCanvasAdapter,
} from "@anvilkit/plugin-canvas-studio";
import "@anvilkit/canvas-editor/styles.css";

const canvasStudio = createCanvasStudioPlugin({
  adapter: localStorageCanvasAdapter({ namespace: "demo-canvas" }),
  // The editor's image tool resolves to a host asset id; seed it so the
  // returned id always maps to renderable bytes.
  onPickAsset: async () => "host-image",
  seedAssets: {
    "host-image": { id: "host-image", uri: "data:image/png;base64,…" },
  },
});

export function MyEditor() {
  return <Studio puckConfig={config} plugins={[canvasStudio]} />;
}
```

## 3. 옵션

```ts
interface CreateCanvasStudioPluginOptions {
  readonly adapter: CanvasPersistenceAdapter;
  readonly designBlockComponentType?: string;        // default "DesignBlock"
  readonly canvasSnapshotAdapter?: CanvasSnapshotAdapter;
  readonly onPickAsset?: () => Promise<string>;
  readonly seedAssets?: Readonly<Record<string, CanvasAssetRef>>;
}
```

`CanvasPersistenceAdapter`는 스토리지 계약입니다 — `save(designId, ir)`,
`load(designId)`, `list()`. `inMemoryCanvasAdapter`와
`localStorageCanvasAdapter({ namespace })`는 기본 제공됩니다. 영속적인
스토리지(Postgres, IndexedDB)를 원하는 호스트는 이 인터페이스를 직접
구현합니다.

## 4. design:// 참조

플러그인은 `design://` 에셋 리졸버를 등록하여 Puck 페이지가 디자인 id로
아트보드 미리보기를 참조할 수 있도록 합니다. 자체 파이프라인을 구성하는
호스트를 위해 접두사와 리졸버 팩토리가 내보내져 있습니다:

```ts
import {
  createDesignAssetResolver,
  DESIGN_REFERENCE_PREFIX, // "design://"
} from "@anvilkit/plugin-canvas-studio";
```

## 5. 모드 전환 및 스냅샷

- 헤더 액션(`canvas-studio:toggle`, id는 `MODE_SWITCH_ACTION_ID`)은 Puck 페이지
  편집과 전체 화면 캔버스 오버레이 사이를 전환합니다. 모드 스토어
  (`createCanvasModeStore`)가 활성 모드를 보유합니다.
- `exportCanvasToAsset` / `exportAllArtboards`는 아트보드를 미리보기용으로
  래스터화합니다.
- `canvasSnapshotAdapter`가 제공되면 `createCanvasSnapshotBridge`가 캔버스를
  Version History 플러그인에 연결합니다(제공되지 않으면 프로세스 내 어댑터가
  사용됩니다).

<Callout type="info" title="상태(alpha, v0.1.3)">
API는 `1.0` 이전에 변경될 수 있습니다. `@anvilkit/core`, `@anvilkit/design-block`,
`@anvilkit/plugin-asset-manager`는 이 플러그인의 직접 의존성으로 함께
배포됩니다.
</Callout>
