description: "Puck <Studio> 셸 안에 전체 화면 캔버스 편집 표면을 마운트하고, 호스트 어댑터를 통해 CanvasIR을 영속화하며, @anvilkit/plugin-canvas-studio로 design:// 미리보기를 해석합니다."
---

`@anvilkit/plugin-canvas-studio`는 Puck 기반의 `<Studio>` 셸 안에 전체 화면
**Canvas Studio** 편집 표면을 마운트합니다. 헤더 액션으로 Puck 페이지 모드와
캔버스 오버레이 사이를 전환합니다. 닫을 때 플러그인은 호스트가 제공한
어댑터를 통해 `CanvasIR`을 영속화하고, 각 아트보드를 미리보기로
래스터화하며, 원본 Puck `DesignBlock`을 패치하여 페이지가 그 썸네일을
렌더링하도록 합니다. 또한 디자인 블록을 위한 레이어 빠른 추가 항목과,
페이지가 디자인 id로 아트보드 미리보기를 참조할 수 있는 `design://` 에셋
리졸버도 등록합니다.

## 1. 설치

```sh
pnpm add @anvilkit/plugin-canvas-studio @anvilkit/canvas-core @anvilkit/canvas-editor react react-dom @puckeditor/core
```

피어 의존성: `@anvilkit/canvas-core`, `@anvilkit/canvas-editor`,
`@puckeditor/core ^0.22.0`, `react >=19.0.0`,
`react-dom >=19.0.0`.

<Callout type="warn" title="에디터 스타일시트 가져오기">
Tailwind 유틸리티와 부모 문서 CSS는 캔버스에 **도달하지 않습니다** — 에디터는
자체 완결형 스타일시트를 함께 제공합니다. 호스트는 그것을 한 번 가져와야
하며, 예를 들어 앱 진입점에서:

```ts
import "@anvilkit/canvas-editor/styles.css";
```
</Callout>

## 2. 최소 호스트

```ts
import { Studio } from "@anvilkit/core";
import {
  createCanvasStudioPlugin,
  localStorageCanvasAdapter,
} from "@anvilkit/plugin-canvas-studio";
import "@anvilkit/canvas-editor/styles.css";

const canvasStudio = createCanvasStudioPlugin({
  adapter: localStorageCanvasAdapter({ namespace: "demo-canvas" }),
  // The editor's image tool resolves to a host asset id; seed it so the
  // returned id always maps to renderable bytes.
  onPickAsset: async () => "host-image",
  seedAssets: {
    "host-image": { id: "host-image", uri: "data:image/png;base64,…" },
  },
});

export function MyEditor() {
  return <Studio puckConfig={config} plugins={[canvasStudio]} />;
}
```

## 3. 옵션

```ts
interface CreateCanvasStudioPluginOptions {
  readonly adapter: CanvasPersistenceAdapter;
  readonly designBlockComponentType?: string;        // default "DesignBlock"
  readonly canvasSnapshotAdapter?: CanvasSnapshotAdapter;
  readonly onPickAsset?: () => Promise<string>;
  readonly seedAssets?: Readonly<Record<string, CanvasAssetRef>>;
}
```

`CanvasPersistenceAdapter`는 스토리지 계약입니다 — `save(designId, ir)`,
`load(designId)`, `list()`. `inMemoryCanvasAdapter`와
`localStorageCanvasAdapter({ namespace })`는 기본 제공됩니다. 영속적인
스토리지(Postgres, IndexedDB)를 원하는 호스트는 이 인터페이스를 직접
구현합니다.

## 4. design:// 참조

플러그인은 `design://` 에셋 리졸버를 등록하여 Puck 페이지가 디자인 id로
아트보드 미리보기를 참조할 수 있도록 합니다. 자체 파이프라인을 구성하는
호스트를 위해 접두사와 리졸버 팩토리가 내보내져 있습니다:

```ts
import {
  createDesignAssetResolver,
  DESIGN_REFERENCE_PREFIX, // "design://"
} from "@anvilkit/plugin-canvas-studio";
```

## 5. 모드 전환 및 스냅샷

- 헤더 액션(`canvas-studio:toggle`, id는 `MODE_SWITCH_ACTION_ID`)은 Puck 페이지
  편집과 전체 화면 캔버스 오버레이 사이를 전환합니다. 모드 스토어
  (`createCanvasModeStore`)가 활성 모드를 보유합니다.
- `exportCanvasToAsset` / `exportAllArtboards`는 아트보드를 미리보기용으로
  래스터화합니다.
- `canvasSnapshotAdapter`가 제공되면 `createCanvasSnapshotBridge`가 캔버스를
  Version History 플러그인에 연결합니다(제공되지 않으면 프로세스 내 어댑터가
  사용됩니다).

## 6. 헤드리스 문서(`@anvilkit/canvas-core`)

데이터 계층은 React나 Konva에 의존하지 않습니다 — 서버, Worker, 테스트 등
어디서든 문서를 생성·편집·검사할 수 있습니다:

```ts
import {
  applyCommand,
  applyCommands,
  createCanvasIR,
  createPage,
  createRect,
  insertNode,
} from "@anvilkit/canvas-core";

// 한 페이지짜리 문서를 만듭니다.
const page = createPage({ id: "p1" });
let ir = createCanvasIR({ id: "doc-1", title: "Launch poster", pages: [page] });
ir = insertNode(ir, {
  parentId: page.root.id,
  node: createRect({ id: "hero", bounds: { width: 640, height: 360 } }),
});

// 모든 변경은 되돌릴 수 있는 커맨드입니다: `applyCommand`는 다음 문서와
// 실행 취소 스택용의 간결한 역커맨드를 반환합니다.
const moved = applyCommand(ir, {
  type: "node.move",
  nodeId: "hero",
  from: { x: 0, y: 0 },
  to: { x: 120, y: 80 },
});
ir = moved.ir;
const undo = moved.inverse;

// 배치는 전부 적용되거나 전부 취소되며(all-or-nothing), 하나의 실행 취소
// 단계로 반전되고, 자동 저장/협업 구독자를 위한 세분화된 변경 레코드를
// 생성합니다.
const batch = applyCommands(
  ir,
  [{ type: "node.move", nodeId: "hero", from: { x: 120, y: 80 }, to: { x: 0, y: 0 } }],
  { label: "reset layout" },
);
ir = batch.ir; // batch.inverse는 배치 전체를 취소, batch.changes는 변경 목록
```

## 7. SVG / PDF 내보내기

```ts
import {
  serializeDocumentToPdf,
  serializePageToSvg,
} from "@anvilkit/canvas-core";

// SVG: 페이지 단위(id 또는 인덱스), 비동기, 구조화된 warnings 계약 —
// 충실도 격차는 throw가 아니라 `warnings`로 보고됩니다.
const { svg, warnings } = await serializePageToSvg(ir, "p1", {
  images: "embed", // 이미지 에셋을 data: URI로 인라인
  pretty: true,
});
if (warnings.length) console.warn(warnings);

// PDF: 래스터 임베드 — 미리 렌더링한 페이지 비트맵(예: 에디터의
// `rasterizePage`)을 전달하면 캔버스 페이지마다 한 페이지의 PDF가 생성됩니다.
const { pdf } = await serializeDocumentToPdf(ir, {
  rasters: [{ pageId: "p1", image: pngBytesOrDataUrl }],
  title: "Launch poster",
});
```

호스트 내보내기 메뉴용으로는 `@anvilkit/plugin-export-canvas`가 이
직렬화기를 `canvas-json` / `canvas-svg` / `canvas-pdf` / `canvas-png`
내보내기 형식으로 패키징해 제공합니다.

## 8. 에디터 단독 마운트(`@anvilkit/canvas-editor`)

전체 워크스페이스 셸은 `<Studio>` 밖에서도 마운트할 수 있습니다 — Konva는
클라이언트 사이드에서만 렌더링되므로 Next.js 호스트에서는 `ssr: false`로
로드하세요:

```tsx
"use client";

import { CanvasWorkspace } from "@anvilkit/canvas-editor";
import "@anvilkit/canvas-editor/styles.css";

export function DesignSurface({ initial }: { initial: CanvasIR }) {
  return (
    <CanvasWorkspace
      storeId="design-1" // 문서별로 안정적으로 유지 — 레일/독 UI 상태 보존
      title="Launch poster"
      initialIR={initial}
      onChange={(ir) => save(ir)} // 코스 그레인. 세분화된 변경은 `onChanges`
      onBack={() => close()}
    />
  );
}
```

`CanvasWorkspace`는 모든 `CanvasStudio` 프로퍼티(`brandKit`, `messages`,
`extensions`, `onStageReady` 등)와 셸 시임(`headerPlugins`, `dockItems`,
`panels`, `inspector`)을 받습니다. 전체 API는
[canvas-editor API 레퍼런스](/api/canvas-editor)를 참고하세요.

## 9. 아키텍처와 경계

캔버스 서브시스템은 엄격한 계층 스택입니다 — 각 계층은 아래 계층만
사용합니다:

```
host app (demo, docs, your product)
  └─ @anvilkit/core <Studio> ── @anvilkit/plugin-canvas-studio (overlay, DesignBlock
       │                        round-trip, design:// resolver, persistence adapters)
       └─ @anvilkit/plugin-export-canvas (host export formats)
            └─ @anvilkit/canvas-editor (React 19 + Konva UI: workspace, tools,
                 panels, history, a11y, optional ./collab subpath)
                 └─ @anvilkit/canvas-core (headless IR: schema, commands,
                      geometry, serializers — deps: zod + pdf-lib ONLY)
```

각 패키지가 강제하며 CI 게이트가 보장하는 규칙:

- `canvas-core`는 React, Konva, DOM을 절대 임포트하지 않습니다 — 릴리스
  게이트가 이를 스캔합니다. 모든 상태 변경은 되돌릴 수 있는
  `CanvasCommand`입니다.
- IR 검증은 알 수 없는 필드를 보존합니다(협업 및 최신 피어와의 전방
  호환성). 스키마 마이그레이션은 마이그레이션 레지스트리를 거칩니다.
- 내장 노드 종류와 커맨드 종류는 확장으로 가릴 수 없습니다.
- yjs는 선택 사항이며 `./collab` 서브패스로 한정됩니다.
- 번들 예산: `canvas-core` ≤ 80 KB gzip, `canvas-editor` ≤ 400 KB gzip
  (각 패키지의 `.size-limit.json`).

**기존 에디터 UI가 기준선입니다.** 워크스페이스 셸(헤더, 툴 레일, 독,
페이지 스트립), 패널(레이어, 인스펙터, 브랜드, 엘리먼트), 10가지 도구,
키보드 접근성, 실행 취소/다시 실행은 이미 제공됩니다 — 다시 만들지 말고
이 위에 구축하세요. API 레퍼런스:
[canvas-core](/api/canvas-core) · [canvas-editor](/api/canvas-editor) ·
[plugin-canvas-studio](/api/plugins/plugin-canvas-studio) ·
[plugin-export-canvas](/api/plugins/plugin-export-canvas).

<Callout type="info" title="상태(alpha)">
API는 `1.0` 이전에 변경될 수 있습니다 — 버전은 각 패키지의 `package.json`을
따릅니다. `@anvilkit/core`, `@anvilkit/design-block`,
`@anvilkit/plugin-asset-manager`는 이 플러그인의 직접 의존성으로 함께
배포됩니다.
</Callout>
