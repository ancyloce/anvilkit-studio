---
title: Multi-page management
description: Capability-gated row actions, page settings, SEO, search, and drag-reorder for the Studio pages panel.
---

The pages panel in `<Studio>` is driven by a single host-supplied
`StudioPagesSource`. Every mutation affordance — rename, delete,
duplicate, drag-reorder, settings, SEO — is an **optional** callback
on that interface. When the host implements a callback, the matching
UI affordance appears in `PageRow`. When it doesn't, the affordance is
silently hidden. No host changes are forced; the same source that
shipped pre-v0.2 continues to compile and render unchanged.

## 1. What you get

| Feature | Callback | Affordance | Hidden when… |
| --- | --- | --- | --- |
| Rename | `onRename` | Inline `Input` on the row | callback undefined **or** `page.locked === true` |
| Delete | `onDelete` | Row menu → confirm dialog | callback undefined **or** `page.locked === true` |
| Duplicate | `onDuplicate` | Row menu item | callback undefined |
| Page settings | `onUpdateSettings` | Settings dialog | callback undefined |
| SEO metadata | `onUpdateSettings` | SEO section inside the settings dialog | callback undefined |
| Drag-reorder | `onReorder` | Drag handle on row hover; Space + Arrow keyboard cycle | callback undefined (handle inert) |
| Search / filter | *(none — UI only)* | Header `Input` + filtered list | never — always available |

If a row would render no actions at all, the overflow trigger itself
is suppressed.

## 2. Contract surface

```ts
import type {
  StudioPage,
  StudioPagesSource,
} from "@anvilkit/core/types";

interface StudioPagesSource {
  // existing — unchanged from earlier versions
  list(): readonly StudioPage[] | Promise<readonly StudioPage[]>;
  subscribe?(listener: () => void): () => void;
  onSelect?(pageId: string): void;
  onCreate?(input: StudioPageCreateInput): void | Promise<void>;

  // new — all optional, capability-gated
  onRename?(input: StudioPageRenameInput): void | Promise<void>;
  onDelete?(pageId: string): void | Promise<void>;
  onDuplicate?(pageId: string): void | Promise<StudioPage | void>;
  onReorder?(input: StudioPageReorderInput): void | Promise<void>;
  onUpdateSettings?(input: StudioPageSettingsInput): void | Promise<void>;
}
```

The full interface (including input shapes) is generated under
[`/api/core/interfaces/StudioPagesSource`](/api/core/interfaces/StudioPagesSource/).

<Callout type="info" title="No optimistic mutation">
Every callback resolves → the host mutates its registry → the host
fires its `subscribe()` listener → `PagesPanel` re-runs `list()`.
The UI never mutates ahead of the host. One documented exception:
`onDuplicate` may return the created `StudioPage` so the UI can
pre-select it before the round-trip lands.
</Callout>

## 3. Locked pages

`StudioPage.locked` is the explicit signal a host uses to mark a page
as non-renamable and non-deletable. When set:

- The `Rename` and `Delete` menu items are suppressed on that row,
  regardless of whether `onRename` / `onDelete` are implemented.
- The row shows the Home icon (used to be a `page.id === "home"`
  string heuristic in core — now host-controlled).
- `Duplicate` and `Settings` are **not** suppressed by `locked` —
  hosts can still allow duplicating the locked page or editing its
  metadata.

The demo source marks the home page as `{ id: "home", title: "Home",
locked: true }` — see `apps/studio/lib/demo-pages-source.ts` for a
reference implementation.

## 4. Brand tokens

`PageRow` and the dialogs are themed via a scoped
`--ak-pages-*` token surface, primitive + semantic tiers. Semantic
tokens chain through to `--ak-studio-*`, then to the host's shadcn
vars:

```css
.ak-pages-panel {
  --ak-pages-fg: var(--ak-studio-fg, var(--foreground));
  --ak-pages-muted: var(--ak-studio-muted, var(--muted));
  --ak-pages-accent: var(--ak-pages-accent-primitive);
  --ak-pages-danger-bg: var(--ak-pages-danger-primitive);
  /* … */
}
```

The full token surface lives in
`packages/core/src/studio/layout/sidebar/modules/layer/pages-tokens.css`.
Dark mode overrides primitives only; semantic tokens inherit. Hosts
that already supply `--ak-studio-*` or shadcn theme vars get the
right look without any extra wiring.

## 5. End-to-end reference

A minimal source that implements every callback, modelled on the demo:

```ts
import type {
  StudioPage,
  StudioPagesSource,
  StudioPageRenameInput,
  StudioPageReorderInput,
  StudioPageSettingsInput,
} from "@anvilkit/core/types";

export function createMyPagesSource(): StudioPagesSource {
  const pages: StudioPage[] = [
    { id: "home", title: "Home", locked: true },
    { id: "about", title: "About", path: "/about", route: true },
  ];
  let activeId = pages[0]?.id ?? "";
  const listeners = new Set<() => void>();
  const notify = () => {
    for (const fn of listeners) fn();
  };

  return {
    list: () => pages.map((p) => ({ ...p, active: p.id === activeId })),
    subscribe(fn) {
      listeners.add(fn);
      return () => listeners.delete(fn);
    },
    onSelect(id) {
      activeId = id;
      notify();
    },
    onRename({ id, title }: StudioPageRenameInput) {
      const p = pages.find((x) => x.id === id);
      if (p) {
        p.title = title;
        notify();
      }
    },
    onDelete(id) {
      const i = pages.findIndex((p) => p.id === id);
      if (i >= 0) {
        pages.splice(i, 1);
        notify();
      }
    },
    async onDuplicate(id): Promise<StudioPage | void> {
      const src = pages.find((p) => p.id === id);
      if (!src) return;
      const copy: StudioPage = {
        ...src,
        id: `${id}-copy-${Date.now()}`,
        title: `${src.title} (copy)`,
        locked: undefined,
      };
      pages.push(copy);
      notify();
      return copy; // pre-select on the UI side
    },
    onReorder({ id, toIndex }: StudioPageReorderInput) {
      const from = pages.findIndex((p) => p.id === id);
      if (from < 0) return;
      const [moved] = pages.splice(from, 1);
      if (moved) pages.splice(toIndex, 0, moved);
      notify();
    },
    onUpdateSettings(input: StudioPageSettingsInput) {
      const p = pages.find((x) => x.id === input.id);
      if (!p) return;
      Object.assign(p, input);
      notify();
    },
  };
}
```

Drop the source into `<Studio pages={createMyPagesSource()} … />` and
every affordance lights up.
