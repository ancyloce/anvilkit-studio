`@anvilkit/plugin-canvas-studio` 在基于 Puck 的 `<Studio>` 外壳内挂载一个全屏的
**Canvas Studio** 编辑界面。一个头部操作可在 Puck 页面模式与画布叠加层之间切换；
关闭时，插件通过宿主提供的适配器持久化 `CanvasIR`，将每个画板栅格化为一张预览，
并修补发起的 Puck `DesignBlock`，使页面渲染该缩略图。它还为设计块注册了一个图层
快速添加项，以及一个 `design://` 资源解析器，使页面可以通过设计 id 引用画板预览。

## 1. 安装

```sh
pnpm add @anvilkit/plugin-canvas-studio @anvilkit/canvas-core @anvilkit/canvas-editor react react-dom @puckeditor/core
```

对等依赖：`@anvilkit/canvas-core`、`@anvilkit/canvas-editor`、
`@puckeditor/core ^0.22.1`、`react >=19.0.0`、
`react-dom >=19.0.0`。

<Callout type="warn" title="导入编辑器样式表">
Tailwind 工具类和父文档 CSS **不会**到达画布——编辑器自带一个独立完整的样式表。
宿主必须导入它一次，例如在应用入口处：

```ts
import "@anvilkit/canvas-editor/styles.css";
```
</Callout>

## 2. 最小宿主

```ts
import { Studio } from "@anvilkit/core";
import {
  createCanvasStudioPlugin,
  localStorageCanvasAdapter,
} from "@anvilkit/plugin-canvas-studio";
import "@anvilkit/canvas-editor/styles.css";

const canvasStudio = createCanvasStudioPlugin({
  adapter: localStorageCanvasAdapter({ namespace: "demo-canvas" }),
  // The editor's image tool resolves to a host asset id; seed it so the
  // returned id always maps to renderable bytes.
  onPickAsset: async () => "host-image",
  seedAssets: {
    "host-image": { id: "host-image", uri: "data:image/png;base64,…" },
  },
});

export function MyEditor() {
  return <Studio puckConfig={config} plugins={[canvasStudio]} />;
}
```

## 3. 选项

```ts
interface CreateCanvasStudioPluginOptions {
  readonly adapter: CanvasPersistenceAdapter;
  readonly designBlockComponentType?: string;        // default "DesignBlock"
  readonly canvasSnapshotAdapter?: CanvasSnapshotAdapter;
  readonly onPickAsset?: () => Promise<string>;
  readonly seedAssets?: Readonly<Record<string, CanvasAssetRef>>;
}
```

`CanvasPersistenceAdapter` 是存储契约——`save(designId, ir)`、
`load(designId)` 和 `list()`。`inMemoryCanvasAdapter` 和
`localStorageCanvasAdapter({ namespace })` 开箱即用；想要持久化
存储（Postgres、IndexedDB）的宿主需自行实现该接口。

## 4. design:// 引用

插件注册了一个 `design://` 资源解析器，使 Puck 页面可以通过设计 id 引用
某个画板的预览。前缀和解析器工厂已导出，供搭建自有流水线的宿主使用：

```ts
import {
  createDesignAssetResolver,
  DESIGN_REFERENCE_PREFIX, // "design://"
} from "@anvilkit/plugin-canvas-studio";
```

## 5. 模式切换与快照

- 头部操作（`canvas-studio:toggle`，id 为 `MODE_SWITCH_ACTION_ID`）在
  Puck 页面编辑与全屏画布叠加层之间切换；模式存储
  （`createCanvasModeStore`）保存当前激活的模式。
- `exportCanvasToAsset` / `exportAllArtboards` 将画板栅格化为预览。
- 当提供了 `canvasSnapshotAdapter` 时，`createCanvasSnapshotBridge` 将画布
  接入 Version History 插件（否则使用进程内适配器）。

## 6. 无头文档（`@anvilkit/canvas-core`）

数据层不依赖 React 或 Konva——可在任意环境（服务端、Worker、测试）中构建、
编辑和检查文档：

```ts
import {
  applyCommand,
  applyCommands,
  createCanvasIR,
  createPage,
  createRect,
  insertNode,
} from "@anvilkit/canvas-core";

// 构建单页文档。
const page = createPage({ id: "p1" });
let ir = createCanvasIR({ id: "doc-1", title: "Launch poster", pages: [page] });
ir = insertNode(ir, {
  parentId: page.root.id,
  node: createRect({ id: "hero", bounds: { width: 640, height: 360 } }),
});

// 每次变更都是可逆命令：`applyCommand` 返回新文档以及一个紧凑的
// 逆命令，供你的撤销栈使用。
const moved = applyCommand(ir, {
  type: "node.move",
  nodeId: "hero",
  from: { x: 0, y: 0 },
  to: { x: 120, y: 80 },
});
ir = moved.ir;
const undo = moved.inverse;

// 批量命令以"全有或全无"方式应用，作为单个撤销步骤反转，
// 并派生细粒度变更记录供自动保存 / 协作订阅方使用。
const batch = applyCommands(
  ir,
  [{ type: "node.move", nodeId: "hero", from: { x: 120, y: 80 }, to: { x: 0, y: 0 } }],
  { label: "reset layout" },
);
ir = batch.ir; // batch.inverse 可整体撤销；batch.changes 列出增量
```

## 7. 导出 SVG 与 PDF

```ts
import {
  serializeDocumentToPdf,
  serializePageToSvg,
} from "@anvilkit/canvas-core";

// SVG：按页导出（页 id 或索引），异步，带结构化 warnings 契约——
// 保真度缺口以 `warnings` 报告而非抛错。
const { svg, warnings } = await serializePageToSvg(ir, "p1", {
  images: "embed", // 将图像资产内联为 data: URI
  pretty: true,
});
if (warnings.length) console.warn(warnings);

// PDF：栅格嵌入——提供预渲染的页面位图（例如来自编辑器的
// `rasterizePage`），每个画布页生成一页 PDF。
const { pdf } = await serializeDocumentToPdf(ir, {
  rasters: [{ pageId: "p1", image: pngBytesOrDataUrl }],
  title: "Launch poster",
});
```

面向宿主导出菜单，`@anvilkit/plugin-export-canvas` 将这些序列化器封装为
现成的 `canvas-json` / `canvas-svg` / `canvas-pdf` / `canvas-png` 导出格式。

## 8. 独立挂载编辑器（`@anvilkit/canvas-editor`）

完整的工作区外壳也可以在 `<Studio>` 之外挂载——Konva 仅在客户端渲染，
因此在 Next.js 宿主中请使用 `ssr: false` 加载：

```tsx
"use client";

import { CanvasWorkspace } from "@anvilkit/canvas-editor";
import "@anvilkit/canvas-editor/styles.css";

export function DesignSurface({ initial }: { initial: CanvasIR }) {
  return (
    <CanvasWorkspace
      storeId="design-1" // 每个文档保持稳定——保存侧栏/面板 UI 状态
      title="Launch poster"
      initialIR={initial}
      onChange={(ir) => save(ir)} // 粗粒度；`onChanges` 提供细粒度增量
      onBack={() => close()}
    />
  );
}
```

`CanvasWorkspace` 接受全部 `CanvasStudio` 属性（`brandKit`、`messages`、
`extensions`、`onStageReady` 等）以及外壳插槽（`headerPlugins`、`dockItems`、
`panels`、`inspector`）。完整 API 见
[canvas-editor API 参考](/api/canvas-editor)。

## 9. 架构与边界

画布子系统是严格分层的栈——每一层只依赖其下层：

```
host app (demo, docs, your product)
  └─ @anvilkit/core <Studio> ── @anvilkit/plugin-canvas-studio (overlay, DesignBlock
       │                        round-trip, design:// resolver, persistence adapters)
       └─ @anvilkit/plugin-export-canvas (host export formats)
            └─ @anvilkit/canvas-editor (React 19 + Konva UI: workspace, tools,
                 panels, history, a11y, optional ./collab subpath)
                 └─ @anvilkit/canvas-core (headless IR: schema, commands,
                      geometry, serializers — deps: zod + pdf-lib ONLY)
```

各包强制执行（且由 CI 门禁保障）的规则：

- `canvas-core` 绝不导入 React、Konva 或 DOM——其发布门禁会扫描此项。
  所有状态变更都是可逆的 `CanvasCommand`。
- IR 校验保留未知字段（为协作与更新版本的对端保持前向兼容）；
  Schema 迁移必须通过迁移注册表。
- 内置节点类型与命令类型不可被扩展覆盖。
- yjs 是可选依赖，仅限 `./collab` 子路径。
- 体积预算：`canvas-core` ≤ 80 KB gzip，`canvas-editor` ≤ 400 KB gzip
  （各包内的 `.size-limit.json`）。

**既有编辑器界面即是基线。** 工作区外壳（头部、工具栏、侧栏、页面条）、
各面板（图层、检查器、品牌、元素）、十个工具、键盘无障碍支持以及撤销/重做
均已内置——请在这些界面之上构建，而不是重造它们。API 参考：
[canvas-core](/api/canvas-core) · [canvas-editor](/api/canvas-editor) ·
[plugin-canvas-studio](/api/plugins/plugin-canvas-studio) ·
[plugin-export-canvas](/api/plugins/plugin-export-canvas)。

<Callout type="info" title="状态（alpha）">
API 在 `1.0` 之前可能会变更——版本以各包的 `package.json` 为准。
`@anvilkit/core`、`@anvilkit/design-block` 和
`@anvilkit/plugin-asset-manager` 作为此插件的直接依赖一同发布。
</Callout>
