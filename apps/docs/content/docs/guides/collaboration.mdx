`@anvilkit/collab-ui` ships a single factory that gives a Studio host
the full realtime collaboration experience: presence cursors, conflict
diagnostics, a collaborator avatar stack in the header, and Yjs CRDT
data sync — all behind one `plugins={[...]}` entry, no React wrapper.

## 1. Install

```sh
# the factory + its yjs peer, plus a WebSocket provider for managed mode
pnpm add @anvilkit/collab-ui @anvilkit/plugin-collab-yjs @hocuspocus/provider
```

`@anvilkit/plugin-collab-yjs` is a peer of `@anvilkit/collab-ui` (the
factory composes the two packages together). The provider
(`@hocuspocus/provider`, or `y-websocket`) is an **optional** peer needed
only for managed mode (§ 2) — omit it if you bring your own transport or
run single-tab in-memory. Hosts that want the adapter without React UI can
install only the yjs plugin — see § 4 below.

## 2. Integration recipe

Set **one** value — the WebSocket relay URL. The plugin owns the entire
transport: the `Y.Doc`, the `Awareness` channel, the provider, the
connection-status bridge, and teardown. `room` defaults to
`"anvilkit-default-room"`, `provider` to `"hocuspocus"`, and `self` to an
auto-generated anonymous identity, so this is genuinely the whole recipe.

```tsx
import { Studio } from "@anvilkit/core";
import { createCollabPlugin } from "@anvilkit/collab-ui";

<Studio
	puckConfig={myConfig}
	plugins={[
		createCollabPlugin({ websocketUrl: "ws://localhost:1234", puckConfig: myConfig }),
	]}
/>;
```

<Callout type="info" title="Install the provider">
	Managed mode loads the WebSocket provider **lazily** (it never reaches your
	initial bundle), so install the one you use — both are optional peers:
	`@hocuspocus/provider` (the default backend) or `y-websocket` (when you set
	`provider: "y-websocket"`). If you set `websocketUrl` without installing the
	provider, the sync indicator shows a clear `error` status and
	`onConnectionError` fires — never a cryptic bundler failure.
</Callout>

### Bring your own transport

Already managing the `Y.Doc` / `Awareness` / provider yourself (a custom
backend, a shared doc, or a host that tees connection status into its own UI)?
Pass them directly and the factory leaves transport entirely to you — fully
backward compatible with pre-`websocketUrl` callers:

```tsx
import { Doc as YDoc } from "yjs";
import { Awareness } from "y-protocols/awareness";

const doc = new YDoc();
const awareness = new Awareness(doc);

createCollabPlugin({
	doc,
	awareness,
	self: { id: "alice", displayName: "Alice", color: "#f43f5e" },
	puckConfig: myConfig,
	onIdentityChange: (next) => {
		// Mirror identity changes (e.g. from the settings popover)
		// back into your auth/profile store.
		updateProfile(next);
	},
});
```

When both `doc` and `websocketUrl` are passed, `doc` (BYO) wins and a one-time
dev warning names the ignored field.

The plugin contributes:

| Contribution        | What you see                                                  |
| ------------------- | ------------------------------------------------------------- |
| `hooks`             | Puck data ↔ Y.Doc CRDT sync (Puck stays the source of truth). |
| Provider            | `<CollabUIProvider>` wrapping Studio so `useCollabContext` works everywhere. |
| Canvas overlay      | `<PresenceLayer>` — remote peer cursors and selection rings.  |
| Notifications overlay | `<ConflictNoticeCenter>` — toast notifications on overlap. |
| Collaborators slot  | `<PeerAvatarStack>` in the header.                            |

## 3. Common knobs

```tsx
createCollabPlugin({
	// Managed-mode transport knobs (all optional; `websocketUrl` is the
	// only field most hosts set)
	websocketUrl: "wss://relay.example.com",
	room: "doc-42", //              default "anvilkit-default-room"
	provider: "y-websocket", //     default "hocuspocus"
	token: authToken, //            forwarded to the relay (auth)
	self: alice, //                 default: auto-generated anonymous identity
	onConnectionError: (err) => toast.error(String(err)),

	// Validate remote IR before dispatching (defense-in-depth)
	validateRemoteIR: (ir) => (isWellFormed(ir) ? ir : null),
	onValidationFailure: (failure) => toast(`Rejected: ${failure.kind}`),

	// RBAC / lock policy applied symmetrically on inbound + outbound
	policy: {
		canEdit: (node, peer) => canUserEditNode(peer, node),
	},
	onPolicyViolation: (violation) => toast(violation.message),

	// Surface outbound transport failures
	onSaveError: (error) => toast(`Save failed: ${error}`),

	// Cross-tab persistence
	persistence: { indexedDb: true, broadcastChannel: true },

	// Opt out of individual UI surfaces
	presence: { enabled: false }, //      hide remote cursors
	notifications: { enabled: false }, // hide conflict toaster
});
```

When the host passes `collaboratorsSlot` on `<Studio>`, that prop wins
over the plugin's slot contribution — the host always has the final
word.

## 4. Power-user path (headless / alt UI)

If you don't need React UI — e.g. a Node sync worker, a server-side IR
processor, or a custom non-React collaborator chrome — import the
low-level primitives from `@anvilkit/plugin-collab-yjs` directly:

```ts
import {
	createCollabDataPlugin,
	createYjsAdapter,
} from "@anvilkit/plugin-collab-yjs";

const adapter = createYjsAdapter({ doc, awareness, peer });
const plugin = createCollabDataPlugin({ adapter, puckConfig });
```

<Callout type="warn" title="Renaming note">
	The data-only factory in `@anvilkit/plugin-collab-yjs` was renamed
	`createCollabPlugin` → `createCollabDataPlugin` to disambiguate from
	the consolidated factory in `@anvilkit/collab-ui`. The legacy alias
	`createCollabPlugin` remains exported for one minor release and
	emits a one-shot `console.warn` on first call.
</Callout>

## 5. Composition with Puck's plugin contract

A Studio plugin can ship **any combination** of Puck-shape fields and
Studio extensions. The consolidated collab plugin uses the Studio
extensions; you can add Puck-shape fields alongside if you want a
Collaborators panel in the plugin rail.

| Plugin contribution | Contract  | Target                                                    |
| ------------------- | --------- | --------------------------------------------------------- |
| `name` / `label` / `icon` | Puck   | Plugin rail identity                                  |
| `render`            | Puck      | Plugin rail panel                                         |
| `overrides`         | Puck (passthrough through Studio) | Inside-`<Puck>` slots          |
| `hooks`             | Studio    | Lifecycle (onInit / onDataChange / onBeforePublish / …)   |
| `providers`         | Studio    | Wrap the entire Studio tree                               |
| `overlays`          | Studio    | Siblings of the editor (canvas / viewport / notifications) |
| `slots`             | Studio    | Named chrome anchors (`collaborators`)                    |

See the [plugin authoring guide](./plugin-authoring.mdx) for the full
plugin contract details.
