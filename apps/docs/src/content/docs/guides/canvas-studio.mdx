import { Aside } from "@astrojs/starlight/components";

`@anvilkit/plugin-canvas-studio` mounts a full-screen **Canvas Studio** editing
surface inside the Puck-based `<Studio>` shell. A header action toggles between
Puck page mode and a canvas overlay; on close the plugin persists the
`CanvasIR` through a host-provided adapter, rasterizes each artboard to a
preview, and patches the originating Puck `DesignBlock` so the page renders the
thumbnail. It also registers a layer quick-add for design blocks and a
`design://` asset resolver so pages can reference artboard previews by design id.

## 1. Install

```sh
pnpm add @anvilkit/plugin-canvas-studio @anvilkit/canvas-core @anvilkit/canvas-editor react react-dom @puckeditor/core
```

Peer dependencies: `@anvilkit/canvas-core`, `@anvilkit/canvas-editor`,
`@puckeditor/core ^0.21.2`, `react >=19.0.0`,
`react-dom >=19.0.0`.

<Aside type="caution" title="Import the editor stylesheet">
Tailwind utilities and parent-document CSS do **not** reach the canvas — the
editor ships its own self-contained stylesheet. The host must import it once,
e.g. in the app entry:

```ts
import "@anvilkit/canvas-editor/styles.css";
```
</Aside>

## 2. Minimal host

```ts
import { Studio } from "@anvilkit/core";
import {
  createCanvasStudioPlugin,
  localStorageCanvasAdapter,
} from "@anvilkit/plugin-canvas-studio";
import "@anvilkit/canvas-editor/styles.css";

const canvasStudio = createCanvasStudioPlugin({
  adapter: localStorageCanvasAdapter({ namespace: "demo-canvas" }),
  // The editor's image tool resolves to a host asset id; seed it so the
  // returned id always maps to renderable bytes.
  onPickAsset: async () => "host-image",
  seedAssets: {
    "host-image": { id: "host-image", uri: "data:image/png;base64,…" },
  },
});

export function MyEditor() {
  return <Studio puckConfig={config} plugins={[canvasStudio]} />;
}
```

## 3. Options

```ts
interface CreateCanvasStudioPluginOptions {
  readonly adapter: CanvasPersistenceAdapter;
  readonly designBlockComponentType?: string;        // default "DesignBlock"
  readonly canvasSnapshotAdapter?: CanvasSnapshotAdapter;
  readonly onPickAsset?: () => Promise<string>;
  readonly seedAssets?: Readonly<Record<string, CanvasAssetRef>>;
}
```

The `CanvasPersistenceAdapter` is the storage contract — `save(designId, ir)`,
`load(designId)`, and `list()`. `inMemoryCanvasAdapter` and
`localStorageCanvasAdapter({ namespace })` ship in-box; hosts wanting durable
storage (Postgres, IndexedDB) implement the interface themselves.

## 4. design:// references

The plugin registers a `design://` asset resolver so a Puck page can reference
an artboard preview by design id. The prefix and resolver factory are exported
for hosts wiring their own pipeline:

```ts
import {
  createDesignAssetResolver,
  DESIGN_REFERENCE_PREFIX, // "design://"
} from "@anvilkit/plugin-canvas-studio";
```

## 5. Mode switch & snapshots

- The header action (`canvas-studio:toggle`, id `MODE_SWITCH_ACTION_ID`) flips
  between Puck page editing and the full-screen canvas overlay; the mode store
  (`createCanvasModeStore`) holds the active mode.
- `exportCanvasToAsset` / `exportAllArtboards` rasterize artboards for previews.
- `createCanvasSnapshotBridge` wires the canvas into the Version History plugin
  when a `canvasSnapshotAdapter` is supplied (otherwise an in-process adapter is
  used).
