`@anvilkit/plugin-version-history` is a first-party headless `StudioPlugin`.
Snapshot persistence is delegated to a host-provided `SnapshotAdapter`, so the
plugin ships no I/O of its own — only the diff/apply engine, two header actions,
optional UI primitives, and reference adapters for tests and demos.

## 1. Install

```sh
pnpm add @anvilkit/plugin-version-history
```

Peer dependencies:

- `@puckeditor/core ^0.21.3`
- `react >=19.0.0`
- `react-dom >=19.0.0`

No transport or storage dependencies — the host owns persistence end-to-end.

Subpath exports:

- `@anvilkit/plugin-version-history` — factory, reference adapters, diff/apply engine, types.
- `@anvilkit/plugin-version-history/ui` — `VersionHistoryUI`, `SaveSnapshotButton`, `SnapshotList`, `SnapshotHistoryModal`, `DiffView`.
- `@anvilkit/plugin-version-history/testing` — the `runAdapterContract` shared test suite.

## 2. Minimal host

```ts
import { Studio } from "@anvilkit/core";
import {
  createVersionHistoryPlugin,
  localStorageAdapter,
} from "@anvilkit/plugin-version-history";

const versionHistory = createVersionHistoryPlugin({
  adapter: localStorageAdapter({ namespace: "my-app-history" }),
  maxSnapshots: 50,
});

export function MyEditor() {
  return <Studio puckConfig={config} plugins={[versionHistory]} />;
}
```

The plugin contributes two header actions (`version-history:save` and
`version-history:open`) and emits `version-history:save-requested` /
`version-history:open-requested` on the Studio event bus. Handle the open event
by mounting a UI surface — e.g. `<SnapshotHistoryModal>` from the `/ui` subpath.

## 3. Options

```ts
interface CreateVersionHistoryPluginOptions {
  readonly adapter: SnapshotAdapter;
  readonly maxSnapshots?: number; // FIFO eviction; requires adapter.delete
}
```

When `maxSnapshots` is set, the oldest snapshot is automatically deleted once
capacity is reached (this requires the adapter to implement `delete`).

## 4. Adapters

`inMemoryAdapter()` (tests) and `localStorageAdapter({ namespace })` (demos)
ship in-box. A production adapter implements `save` / `list` / `load` and
optionally `delete`:

```ts
import type { SnapshotAdapter } from "@anvilkit/plugin-version-history";

const firestoreAdapter: SnapshotAdapter = {
  save: async (id, record) => { /* … */ },
  list: async () => { /* … */ },
  load: async (id) => { /* … */ },
  delete: async (id) => { /* … */ },
};
```

Validate any custom adapter against the same contract the reference adapters
use, via `runAdapterContract` on the `/testing` subpath.

## 5. Diff engine

`diffIR(a, b)` produces a frozen `IRDiff`; `applyDiff(a, diff)` round-trips back
to `b`; `summarizeDiff(diff)` returns an `IRDiffSummary` for display:

```ts
import { applyDiff, diffIR, summarizeDiff } from "@anvilkit/plugin-version-history";

const diff = diffIR(previous, current);
const summary = summarizeDiff(diff);   // { added, removed, changed, … }
const restored = applyDiff(previous, diff); // deep-equals `current`
```

A malformed apply throws `DiffApplyError`; other failures surface as
`VersionHistoryError` carrying a `VersionHistoryErrorCode`.

</Aside>
