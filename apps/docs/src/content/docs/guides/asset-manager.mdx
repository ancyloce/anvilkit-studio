`@anvilkit/plugin-asset-manager` is a first-party headless `StudioPlugin`. It
ships a working in-memory asset library out of the box — **every option is
optional**. Wire only the `fetch`/data operations you care about (list / remove
/ replace / rename / move / create-folder) and the plugin encapsulates the rest:
validation, an in-memory registry with search, folders, Unsplash, categories &
filtering, IR-time `asset://` resolution, CSP guidance, and an optional React UI.
Every adapter response crosses a strict trust boundary before it is registered.

## 1. Install

```sh
pnpm add @anvilkit/plugin-asset-manager
```

Peer dependencies (already in the typical Studio host):

- `@puckeditor/core ^0.21.3`
- `react >=19.0.0`
- `react-dom >=19.0.0`

Subpath exports:

- `@anvilkit/plugin-asset-manager` — factory, adapters, validation, CSP advisor, errors.
- `@anvilkit/plugin-asset-manager/ui` — React UI components (browser + folders + Unsplash panel).
- `@anvilkit/plugin-asset-manager/adapters/s3` — production `s3PresignedAdapter`.
- `@anvilkit/plugin-asset-manager/providers/unsplash` — the lazy, dependency-free Unsplash provider.
- `@anvilkit/plugin-asset-manager/retry` — `RetryableError` + `withRetry()`.
- `@anvilkit/plugin-asset-manager/testing` — fixtures for downstream plugin tests.

## 2. Zero-config

`createAssetManagerPlugin()` works with **no arguments** — you get an in-memory
library with folders enabled. Uploaded files become page-lifetime `blob:`/`data:`
URLs; nothing is persisted server-side.

```ts
import { Studio } from "@anvilkit/core";
import { createAssetManagerPlugin } from "@anvilkit/plugin-asset-manager";

const assetManager = createAssetManagerPlugin();

export function MyEditor() {
  return <Studio puckConfig={config} plugins={[assetManager]} />;
}
```

For real uploads, pass an `uploader`. `dataUrlUploader` is dev-only (files become
in-memory `data:` URLs); swap in `s3PresignedAdapter` or a custom `UploadAdapter`
for production.

```ts
import { createAssetManagerPlugin } from "@anvilkit/plugin-asset-manager";
import { s3PresignedAdapter } from "@anvilkit/plugin-asset-manager/adapters/s3";

const assetManager = createAssetManagerPlugin({
  uploader: s3PresignedAdapter({
    presignEndpoint: "/api/s3/presign",
    region: "us-east-1",
  }),
});
```

Hoist the factory call to module scope so `<Studio>`'s plugin compile pipeline
doesn't re-run on every render. The default Studio sidebar's **Images** rail
renders the library, folder breadcrumb/tree, source tabs and (when configured)
the Unsplash picker — no extra wiring required.

## 3. Options

Every field is optional. Existing `createAssetManagerPlugin({ uploader })` calls
keep working unchanged.

```ts
interface AssetManagerOptions {
  readonly uploader?: UploadAdapter; // binary ingest; defaults to an in-memory uploader
  readonly dataSource?: AssetDataSource; // host-backed catalog (list / CRUD / folders) — §4
  readonly folders?: boolean | FolderOptions; // folder chrome (default: enabled, in-memory) — §5
  readonly providers?: readonly AssetSourceProvider[]; // extra read-only sources (federated)
  readonly unsplash?: UnsplashSourceOptions; // built-in Unsplash source (proxy-first) — §6
  readonly categories?: readonly AssetCategory[]; // saved-view chips
  readonly facets?: readonly AssetFacetDefinition[]; // custom faceted filters — §7
  // existing optional fields, unchanged:
  readonly maxFileSize?: number;
  readonly acceptedMimeTypes?: readonly string[];
  readonly dataUrlAllowlistOptIn?: boolean; // permit data: URLs (default false)
  readonly allowMixedScriptHostnames?: boolean; // homoglyph guard (default false)
  readonly getThumbnail?: (entry: UploadResult) => string | undefined;
}
```

`uploader` (binary ingest) and `dataSource` (catalog operations) are orthogonal:
`dataSource.replace(id, file)` routes the `File` through `uploader` first, then
hands the result to the catalog. First-time uploads always flow through `uploader`.

## 4. Host-backed data source

To back the library with your own API, supply a `dataSource`. Implement **only**
the methods you need — omitted methods fall back to the in-memory default. The
read/write planes resolve as a unit: if you supply a _partial_ asset (or folder)
method set, the whole plane falls back to in-memory and a one-time warning is
logged.

```ts
const assetManager = createAssetManagerPlugin({
  dataSource: {
    async list(q, signal) {
      return (
        await fetch(`/api/assets?folder=${q.folderId ?? ""}`, { signal })
      ).json();
    },
    async remove(id) {
      await fetch(`/api/assets/${id}`, { method: "DELETE" });
    },
    async rename(id, name) {
      return (
        await fetch(`/api/assets/${id}`, {
          method: "PATCH",
          body: JSON.stringify({ name }),
        })
      ).json();
    },
    async move(id, folderId) {
      await fetch(`/api/assets/${id}`, {
        method: "PATCH",
        body: JSON.stringify({ folderId }),
      });
    },
    async listFolders() {
      return (await fetch("/api/folders")).json();
    },
    async createFolder(p, name) {
      return (
        await fetch("/api/folders", {
          method: "POST",
          body: JSON.stringify({ parentId: p, name }),
        })
      ).json();
    },
  },
});
```

`list(query)` returns both the asset page **and** the current folder tree +
breadcrumb (`folders` / `folderPath`), so a single endpoint drives the whole rail.
Async failures should throw `AssetSourceError` (carries `code`, `retryable`,
`status`, `retryAfterMs`); the sidebar surfaces a retry banner.

## 5. Folders

Folders are **on by default** (`folders: true`). Folder membership is library
metadata only — it never enters the IR, so moving an asset never changes a
rendered page (`asset://<id>` references are folder-independent). Pass
`folders: false` for a flat library, or `folders: { maxDepth, allowMove }` to
configure.

```ts
const assetManager = createAssetManagerPlugin({
  folders: { maxDepth: 4, allowMove: true },
});
```

Removing a folder **reparents** its children to the parent by default (never
cascade-deletes referenced bytes); pass `{ cascade: true }` to a host
`removeFolder` to opt into deletion. Moves are validated all-or-nothing and run a
cycle/`maxDepth` guard.

## 6. Unsplash

Unsplash is built in and **enabled when configured** (proxy-first; the package
embeds no key). Point `proxyEndpoint` at a small server route that injects your
`Client-ID` from a server-only env var so the key never reaches the browser.

```ts
const assetManager = createAssetManagerPlugin({
  uploader: s3PresignedAdapter({
    presignEndpoint: "/api/s3/presign",
    region: "us-east-1",
  }),
  unsplash: { proxyEndpoint: "/api/unsplash", appName: "my-app" },
});
```

The Unsplash code lives behind a lazy subpath import, so it never enters the
headless entry chunk. Multiple themes ship by default (nature, architecture,
business, food, technology, travel, texture, people); override via
`unsplash.themes`. Attribution (`Photo by … on Unsplash` with the mandated UTM
links) and the mandatory download-trigger are enforced by the provider. A
client-side `accessKey` (no `proxyEndpoint`) is **dev-only** — it ships the key
to the browser and logs a one-time warning.

<Aside type="caution" title="No bundled secret">
  The package contains no Unsplash key. Use `proxyEndpoint` in production;
  supply the key to the proxy via a server-only env var (e.g.
  `UNSPLASH_ACCESS_KEY`).
</Aside>

## 7. Categories & filtering

Four orthogonal axes compose with AND: **kind** (derived), **tags** (open),
**folder** (single-parent tree), and **source** (provenance). Hosts append
saved-view `categories` and custom `facets` (local via `valueOf`, or remote) —
they never reconfigure the built-ins. When multiple sources are present the
plugin federates (a rate-limited Unsplash never blanks local results) and pages
with an opaque composite cursor.

## 8. Trust boundary

Every `UploadResult` an adapter returns is run through `validateUploadResult`
before registration: a scheme allowlist (`http`, `https`, `blob` always; `data:`
only with `dataUrlAllowlistOptIn`), a path-traversal guard, and an IDN homoglyph
guard. `javascript:` and `vbscript:` URLs are hard-blocked.

Validation throws `AssetValidationError`; async/folder/provider failures throw
`AssetSourceError`; failed `asset://` resolution at export time throws
`AssetResolutionError`.

## 9. IR-time resolution & CSP

`createIRAssetResolver` / `resolveAssets` turn `asset://<id>` references in the
Page IR into validated URLs at export or render time. `getRequiredCsp` computes
the minimum `connect-src` / `img-src` / `media-src` directives the configured
adapters need:

```ts
import { getRequiredCsp } from "@anvilkit/plugin-asset-manager";

const csp = getRequiredCsp({ adapters: ["s3"] });
```

## 10. Optional React UI

The default Studio sidebar already renders the rich asset chrome. For hosts that
mount their own surface, the `./ui` subpath ships props-driven components:
`UploadButton`, `AssetBrowser`, `AssetCommandPalette`, `MetadataPanel`,
`ReplaceAssetDialog`, `DeleteAssetDialog`, the composite `AssetManagerUI`, the
folder kit (`FolderTree`, `FolderBreadcrumb`, `CreateFolderDialog` /
`RenameFolderDialog`, `DeleteFolderDialog`, `MoveTargetPicker`,
`EmptyFolderState`), and `UnsplashPanel`. Drag-to-folder uses the native HTML5
DnD API (no `@dnd-kit`); every move is also reachable via the keyboard-accessible
`MoveTargetPicker`. The default (`.`) export imports none of them, so headless
consumers pay no UI rendering cost.

<Aside type="note" title="Status (alpha, v0.1.6)">
  Public surface may still shift before `v1.0`. Bundle budgets are enforced in
  CI: headless entry ≤ 8 KB gzip, `./ui` ≤ 12 KB gzip, Unsplash subpath ≤ 4 KB
  gzip. The Unsplash leak gate keeps `api.unsplash.com` / `images.unsplash.com`
  out of the headless chunk.
</Aside>
