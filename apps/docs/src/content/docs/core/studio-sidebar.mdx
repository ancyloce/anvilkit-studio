`<Studio>`'s left sidebar ships a 44 px vertical icon rail plus a
280 px module panel that hosts one of four mutually-exclusive
workspaces: `insert`, `layer`, `image`, `text`. Hosts get the rail +
panel for free; plugins extend any module through five registration
surfaces; consumers can override copy, ratio, or persisted state via
`<Studio messages>` and the per-instance store.

## Modules

| Module | What it does | Persisted state |
| :--- | :--- | :--- |
| `insert` | Drag-and-drop component library ŌĆö sectioned, searchable, grid/list view. Delegates to Puck's `<Puck.Components />`. | `componentViewMode`, `insertSectionsExpanded`, `drawerSearch` (transient) |
| `layer` | Pages list (host-supplied) over the active page's element tree (`<Puck.Outline />`), split by a horizontal drag handle. | `pagesExpanded`, `layerSplitRatio`, `outlineExpanded` |
| `image` | Asset library ŌĆö filter strip (All / Images / Videos / Audio), search, drag-and-drop upload, click-to-insert, per-asset overflow menu. Bridges to `@anvilkit/plugin-asset-manager`. | `assetCategoryFilter` |
| `text` | Copywriting snippet library ŌĆö categories, search, click-to-insert into a selected Text element on the canvas. | `copyCategoryFilter` |

The active module persists per-`<Studio>` instance under
`anvilkit-ui-${storeId}`; supply distinct `storeId` props when
mounting two `<Studio>` instances on the same page.

## Module-switch contract

- Clicking a rail icon for an **inactive** module switches to it.
- Clicking the rail icon for the **already-active** module collapses
  the panel (the rail stays visible).
- Clicking the panel header `├Ś` collapses without changing the
  active module ŌĆö re-clicking any rail icon restores it.
- `Esc` while focus is anywhere inside the panel collapses + returns
  focus to the active rail icon.

## Plugin extension surfaces

Plugins receive a typed `StudioPluginContext` in `register()`.
Each `register*` helper returns an `unregister()` function plugins
should call from their `onDestroy` lifecycle hook.

| Surface | Module | Use it for |
| :--- | :--- | :--- |
| `ctx.registerInsertSection(section)` | `insert` | Add a new collapsible section with a custom predicate over Puck components |
| `ctx.registerLayerQuickAdd(item)` | `layer` (Layers) | Add an entry to the "+" quick-add popover |
| `ctx.registerAssetSource(source)` | `image` | Provide the asset listing + upload + mutation contract |
| `ctx.registerAssetAction(action)` | `image` | Add an entry to the per-asset `ŌĆ”` overflow menu |
| `ctx.registerCopySnippetPack(pack)` | `text` | Provide a set of snippets, optionally per-locale |

The demo in `apps/demo/lib/puck-demo.ts` ships two reference
plugins worth reading before you write your own:

- `demoCopySnippetPlugin` registers an English snippet pack via
  `ctx.registerCopySnippetPack`. Twelve entries split across `basic`
  / `brand` categories ŌĆö useful as a shape reference for
  `StudioCopySnippetPack`.
- `demoLayerQuickAddPlugin` registers a single layer quick-add via
  `ctx.registerLayerQuickAdd`. Reads the live Puck snapshot, then
  dispatches an `insert` action ŌĆö the same shape any plugin
  contributing a quick-add should follow.

The asset-source contract lives in
[`@anvilkit/plugin-asset-manager`](/api/plugin-asset-manager/);
its `studio-asset-source.ts` adapter is the canonical reference for
implementing `StudioAssetSource`.

## i18n catalog

All rail tooltips, panel titles, search placeholders, empty-state
copy, and overflow-menu items resolve through the i18n provider.
The default English catalog ships in
[`editor-i18n-context.tsx`](https://github.com/ancyloce/anvilkit-studio/blob/main/packages/core/src/state/editor-i18n-context.tsx)
under the `studio.module.*` namespace.

Pass overrides via `<Studio messages>`:

```tsx title="app/editor/page.tsx"
<Studio
	puckConfig={config}
	data={data}
	messages={{
		"studio.module.insert.name": "Components",
		"studio.module.layer.name": "Pages",
	}}
/>
```

### Deprecated key aliases

The pre-v1 sidebar shipped two top-level keys ŌĆö `studio.tab.insert`
and `studio.tab.outline` ŌĆö that map to the v1 module names. The
i18n provider keeps an alias map so existing overrides keep
resolving for one release window:

| Legacy key | Resolves to |
| :--- | :--- |
| `studio.tab.insert` | `studio.module.insert.name` |
| `studio.tab.outline` | `studio.module.layer.name` |

Resolution order (precedence high ŌåÆ low): override of the requested
key, override of the legacy alias mapped to the requested key,
default for the requested key, caller-supplied `fallback`, the key
itself. The exhaustive resolution-order test lives at
`packages/core/src/state/__tests__/editor-i18n-context.alias.test.tsx`.

<Aside type="caution" title="Deprecation window">
	The alias map ships for **one release** after the four-module
	sidebar lands. Migrate hosts that override `studio.tab.{insert,outline}`
	to the new `studio.module.*` keys before the next minor.
</Aside>

## CSS variables

The sidebar reuses every existing `--ak-studio-*` token plus two new
layout tokens that the chrome bridges through the iframe theme sync:

| Token | Default | Purpose |
| :--- | :--- | :--- |
| `--ak-studio-rail-width` | `44px` | Width of the always-visible icon rail |
| `--ak-studio-panel-width` | `280px` | Width of the module panel when not collapsed |

Override either at the `:root` (or any ancestor of `<Studio>`) to
re-skin the rail / panel without forking the chrome.

## Migration from the legacy 2-tab shell

`EditorTab` widens from `"insert" | "outline"` to `"insert" |
"layer" | "image" | "text"`. The `editor-ui-store` persist version
bumps to `2`; the hydrate migration rewrites legacy `"outline"` ŌåÆ
`"layer"` once on first load. Consumers narrowing on
`EditorTab === "outline"` must update.

Hosts that wired the legacy `studio.tab.*` keys keep working through
the alias map (one-release window).
