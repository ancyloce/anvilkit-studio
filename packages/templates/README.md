# `@anvilkit/template-*` — first-party seed templates

This directory hosts the 10 seed `AnvilkitTemplate` packages shipped
with `v1.0.0` (task `phase5-017`). Each subdirectory is its own
independently-published npm package under the `@anvilkit/template-*`
scope.

## Contract

Every template package exports a default `AnvilkitTemplate` value
that satisfies the `isAnvilkitTemplate()` guard from
`@anvilkit/core/templates` (ADR 003). A downstream CLI
(`anvilkit init --template <slug>`) resolves the default export,
writes the IR into the scaffolded project, and installs the listed
component packages.

## Layout

```
packages/templates/
├── tsconfig.base.json                     # shared compiler options
├── landing-saas/
│   ├── package.json                       # @anvilkit/template-landing-saas
│   ├── tsconfig.json                      # extends ../tsconfig.base.json
│   ├── README.md                          # one-paragraph blurb
│   └── src/
│       ├── index.ts                       # default export (AnvilkitTemplate)
│       └── page-ir.json                   # the PageIR — committed so diffs are reviewable
└── ... (nine more)
```

## Build

Each package builds via `tsc` (no React, no CSS — just types + JSON).
Turbo picks up the `build` script automatically. Running `pnpm build`
at the repo root builds every template package in parallel with the
rest of the workspace.

## Previews

Preview images (`preview.png`) are generated by
`scripts/capture-template-previews.ts` (Playwright against
`apps/demo/app/puck/render`). The capture step is one-off per
template change — see the script's header for the runbook.
